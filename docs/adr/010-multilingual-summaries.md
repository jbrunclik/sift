# ADR-010: Multilingual summaries

## Status
Accepted

## Context
Sift fetches articles in multiple languages (English tech blogs, Czech news sites). Summaries generated by Gemini default to English, but users may prefer reading summaries in their native language regardless of the source article language.

## Decision
Add a `summary_language` field to `user_profile` (default: `"en"`). The value is passed to the Gemini scoring prompt as an explicit instruction: "Write the summary and explanation in {language}." Currently supported values: `"en"` (English) and `"cs"` (Czech).

The language preference is exposed via `GET/PUT /api/preferences` and selectable in the preferences page. Articles already scored are not retroactively re-summarized; the setting applies to future scoring batches only.

## Consequences
- Summaries feel natural for Czech-speaking users reading Czech sources
- Adding new languages requires no code changes -- just document the new value
- Mixed-language feeds may feel inconsistent if the setting changes mid-batch
- No retroactive re-scoring avoids unnecessary Gemini costs
- The prompt instruction is simple and Gemini handles it reliably for major languages
